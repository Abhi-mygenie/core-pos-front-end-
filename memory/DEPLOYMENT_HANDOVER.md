## 7. Caveats / Open Items for Next Deployment Agent

1. **Backend supervisor program cannot be permanently disabled.** `/etc/supervisor/conf.d/supervisord.conf` is marked `READONLY` (`# READONLY FILE, DO NOT EDIT THIS FILE`) and the `backend` program has `autostart=true`. The current `STOPPED` state was achieved with `supervisorctl stop backend` and will persist only for the life of this supervisord process. **If the container/supervisord restarts, the default FastAPI backend from the repo's `backend/` folder will auto-start on :8001 again.** The frontend does not depend on it (all API traffic goes to `REACT_APP_API_BASE_URL` and `REACT_APP_SOCKET_URL`), so this is cosmetic, but if a hard disable is required, an override file at `/etc/supervisor/conf.d/` (separate filename) setting `autostart=false` would need to be added by a privileged agent.
2. **Banner "Frontend Preview Only. Please wake servers to enable backend functionality."** is shown by the Emergent preview layer because the platform backend (on `REACT_APP_BACKEND_URL`) is not serving app-specific APIs. This is expected for a frontend-only deployment — the real app APIs are hosted at `https://preprod.mygenie.online/` (CRM/POS backend) and `https://presocket.mygenie.online` (sockets), not on the Emergent preview URL.
3. **VAPID key assumption:** The initially supplied `REACT_APP_FIREBASE_VAPID_KEY` in the problem statement was identical to `REACT_APP_FIREBASE_API_KEY` (looked invalid). The user provided a corrected FCM VAPID key during handover (`EvFMTX767yCa...4S-nssyqNG-EIJUu6WIA0MWJaouSUI`) which has been written to `.env`. Next agent should verify web-push registration works end-to-end.
4. **Non-blocking ESLint warning at build time:**

5. `/app/backend/` still contains the default Emergent FastAPI template from the cloned repo. It is unused by this deployment; leaving it intact since brief says no unnecessary code changes.
